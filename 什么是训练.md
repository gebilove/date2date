先看整体代码

def train_one_sample(input_str, target_str, encoder, decoder, encoder_optimizer, decoder_optimizer, criterion):
    input_tensor = tensor_from_string(input_str)
    target_tensor = tensor_from_string(target_str, True)

    encoder_optimizer.zero_grad()
    decoder_optimizer.zero_grad()

    encoder_output, encoder_hidden = encoder(input_tensor)
    decoder_outputs, decoder_hidden = decoder(encoder_hidden, target_tensor)

    logits = decoder_outputs.reshape(-1, vocab_size)
    print("logits", logits.shape)

    target = target_tensor.reshape(-1)
    loss = criterion(logits, target)

    loss.backward()

    encoder_optimizer.step()
    decoder_optimizer.step()

    return loss.item()

---
1. 准备训练数据

input_tensor = tensor_from_string(input_str)
target_tensor = tensor_from_string(target_str, True)

假设输入是：

input_str = "2002-1-23"
target_str = "23/1/2002"

那么：

input_tensor.shape

是：

[1, 9]

含义是：

batch_size = 1
seq_len = 9

即：

"2002-1-23"

被转成 token id。

---
target_tensor = tensor_from_string(target_str, True) 这里 True 表示加 <EOS>。

所以：

target_tensor.shape

是：

[1, 10]

因为：

"23/1/2002"      长度 9
"23/1/2002<EOS>" 长度 10

为什么 target 要加 <EOS>？

因为模型不仅要学会输出：

2 3 / 1 / 2 0 0 2

还要学会：

到这里结束

否则 generate 的时候模型不知道什么时候停。

---
2. 清空旧梯度

encoder_optimizer.zero_grad()
decoder_optimizer.zero_grad()

PyTorch 有一个重要特点：

▎ 默认情况下，梯度是累积的。

也就是说，如果你不清零，第一次训练算出来的梯度会留着，第二次训练的梯度会加到第一次上面。

例如：

第 1 次梯度：0.2
第 2 次梯度：0.3

如果不清零，第二次参数更新用的是：

0.2 + 0.3 = 0.5

这通常不是我们想要的。

所以每次训练一个样本或一个 batch 之前，一般都要：

optimizer.zero_grad()

因为你这里有两个模型：

encoder
decoder

所以两个 optimizer 都要清零。

---
3. 前向传播：模型做预测

encoder_output, encoder_hidden = encoder(input_tensor)

这一步是 Encoder 读输入日期。

输入：

2002-1-23

Encoder 会一个字符一个字符读进去，最后得到：

encoder_hidden

它可以粗略理解为：

▎ Encoder 对整个输入字符串的压缩表示。

形状是：

encoder_hidden.shape == [1, 1, hidden_size]

比如 hidden_size = 32 时：

[1, 1, 32]

三个维度分别是：

num_layers = 1
batch_size = 1
hidden_size = 32

---
然后 Decoder 根据 Encoder 的 hidden 去预测目标日期：

decoder_outputs, decoder_hidden = decoder(encoder_hidden, target_tensor)

因为你传入了 target_tensor，所以 Decoder 使用的是 teacher forcing。

也就是训练时：

第 1 步输入 <SOS>，预测 2
第 2 步输入真实的 2，预测 3
第 3 步输入真实的 3，预测 /
第 4 步输入真实的 /，预测 1
...

注意：训练时下一步输入的是真实答案 token，不是上一步模型自己预测出来的 token。

这样模型早期比较容易训练。

---
Decoder 输出：

decoder_outputs

形状应该是：

[1, 10, vocab_size]

这里：

1 = batch_size
10 = target_len，也就是 "23/1/2002<EOS>" 的长度
vocab_size = 14

所以：

decoder_outputs.shape == [1, 10, 14]

它表示：

▎ 对目标序列的每一个位置，模型都输出了一个长度为 14 的分数向量。

比如第 1 个位置：

decoder_outputs[0, 0]

是一个长度 14 的向量：

[分数<SOS>, 分数<EOS>, 分数0, 分数1, 分数2, ..., 分数/]

这些分数叫 logits。

logits 不是概率，还没有 softmax。

---
4. 整理形状，准备算 loss

logits = decoder_outputs.reshape(-1, vocab_size)

原本：

decoder_outputs.shape == [1, 10, 14]

reshape 后：

logits.shape == [10, 14]

含义是：

10 个预测位置
每个位置对 14 个 token 给出分数

也就是：

位置1：预测 token 分数 14 个
位置2：预测 token 分数 14 个
位置3：预测 token 分数 14 个
...
位置10：预测 token 分数 14 个

---
然后：

target = target_tensor.reshape(-1)

原本：

target_tensor.shape == [1, 10]

reshape 后：

target.shape == [10]

它里面是每个位置的正确答案 token id。

例如目标：

23/1/2002<EOS>

可能对应：

tensor([4, 5, 13, 3, 13, 4, 2, 2, 4, 1])

也就是：

2    -> 4
3    -> 5
/    -> 13
1    -> 3
/    -> 13
2    -> 4
0    -> 2
0    -> 2
2    -> 4
<EOS> -> 1

---
为什么要 reshape？

因为 nn.CrossEntropyLoss() 需要的形状是：

input.shape == [N, C]
target.shape == [N]

其中：

N = 样本数量
C = 类别数量

在你的任务里：

N = 所有时间步数量 = batch_size * target_len = 1 * 10 = 10
C = vocab_size = 14

所以：

logits.shape == [10, 14]
target.shape == [10]

正好符合要求。

---
5. 计算 loss：模型错得有多离谱

loss = criterion(logits, target)

这里的 criterion 通常是：

criterion = nn.CrossEntropyLoss()

它会比较：

模型每个位置输出的 14 个 logits

和：

每个位置真正应该输出的 token id

举个例子，第一个位置目标是 "2"，对应 id 是 4。

模型输出：

logits[0]

可能是：

<SOS>: -0.3
<EOS>:  0.1
0:      0.2
1:     -0.5
2:      2.8   <- 正确答案
3:      0.4
...

如果正确答案 "2" 的分数最高，loss 就小。

如果模型把 "7" 分数打得最高，而正确答案 "2" 分数很低，loss 就大。

---
CrossEntropyLoss 内部做了两件事：

log_softmax + negative log likelihood

所以你不需要自己手动写 softmax。

也就是说，不要这样：

probs = torch.softmax(logits, dim=-1)
loss = criterion(probs, target)

这是不推荐的。

正确就是直接传 logits：

loss = criterion(logits, target)

---
6. 反向传播：计算每个参数应该怎么改

loss.backward()

这是 PyTorch 训练里最核心的一句。

在前向传播时，PyTorch 会自动记录计算图：

embedding 参数
   ↓
RNN 参数
   ↓
Linear 参数
   ↓
logits
   ↓
loss

调用：

loss.backward()

之后，PyTorch 会从 loss 往回推，计算每个参数的梯度。

比如：

decoder.out.weight.grad
encoder.embedding.weight.grad
decoder.rnn.weight_ih_l0.grad

这些 .grad 里会保存：

▎ 为了让 loss 变小，这个参数应该往哪个方向调整。

重要的是：backward() 只负责计算梯度，还没有真正更新参数。

---
7. optimizer.step()：真正更新参数

encoder_optimizer.step()
decoder_optimizer.step()

这一步才会真正修改模型参数。

如果你用的是：

torch.optim.Adam(..., lr=0.01)

那么 optimizer 会根据刚才 .grad 里的梯度，用 Adam 算法更新参数。

概念上可以理解成：

parameter = parameter - learning_rate * gradient

不过 Adam 比这个公式复杂一点，它还会考虑历史梯度的一阶、二阶统计量。

因为你有两个模型：

encoder
decoder

所以要分别：

encoder_optimizer.step()
decoder_optimizer.step()

这样 Encoder 和 Decoder 的参数都会更新。

---
8. 返回 loss 数值

return loss.item()

loss 本身是一个 Tensor，例如：

tensor(2.7351, grad_fn=<NllLossBackward0>)

loss.item() 会把它变成普通 Python 数字：

2.7351

方便你打印、记录、画曲线。

---
这一次训练到底训练了什么？

这一次训练输入是：

2002-1-23

目标是：

23/1/2002<EOS>

模型做的是：

读入 2002-1-23
尝试输出 23/1/2002<EOS>
比较预测和真实答案
根据错误调整 encoder 和 decoder 的参数

一次训练之后，它不一定马上会输出正确结果。
