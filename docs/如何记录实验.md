 对了，如果模型是一个实验科学，我如何高效记录每次参数和实验结果

对，这个思路非常对。训练模型本质上就是实验科学：

假设 -> 设置参数 -> 运行实验 -> 记录结果 -> 分析 -> 下一组实验

如果不记录，很快会变成：

我好像试过 hidden_size=128？
lr 是 0.001 还是 0.01？
那次 accuracy 是多少？
这组权重是哪次训练出来的？

所以你现在应该尽早建立一个轻量实验记录系统。

---
推荐做法：先用 CSV / JSONL，不要一开始上复杂工具

你现在这个 notebook 项目，最适合先用：

experiments.csv

或者：

experiments.jsonl

我更推荐 CSV，因为你可以直接用 Excel / Numbers / pandas 打开看。

---
1. 每次实验要记录什么？

建议至少记录这些字段：

run_id
date
model_type
hidden_size
lr
sample_size
epochs
dropout
optimizer
final_loss
train_correct
train_total
train_acc
test_correct
test_total
test_acc
checkpoint_path
notes

针对你当前任务，可以写成：

run_id：实验编号
model_type：RNN / GRU
hidden_size：隐藏层大小
lr：学习率
sample_size：训练样本数
epochs：训练轮数
dropout：dropout 概率
final_loss：最后一个 epoch 的平均 loss
train_acc：训练集 exact match
test_acc：测试集 exact match
checkpoint_path：对应保存的权重文件
notes：主观观察，比如“年份经常错”“只能背训练集”

---
2. 先准备一个记录函数

可以在 notebook 里加：

import csv
from pathlib import Path
from datetime import datetime

然后定义：

def append_experiment_log(log_path, record):
    log_path = Path(log_path)
    file_exists = log_path.exists()

    with log_path.open("a", newline="", encoding="utf-8") as f:
        writer = csv.DictWriter(f, fieldnames=record.keys())

        if not file_exists:
            writer.writeheader()

        writer.writerow(record)

以后每次训练完，调用一次：

append_experiment_log("experiments.csv", record)

---
3. 给每次实验生成 run_id

可以用时间戳：

from datetime import datetime

run_id = datetime.now().strftime("%Y%m%d_%H%M%S")

例如：

20260625_153012

这样你每次实验都有唯一编号。

---
4. 训练前把超参数集中写在一起

不要把参数散落在 notebook 各处，建议写成一个 config：

config = {
    "model_type": "RNN",
    "hidden_size": 128,
    "lr": 0.001,
    "sample_size": 100,
    "epochs": 1000,
    "dropout": 0.0,
    "optimizer": "Adam",
    "test_size": 100,
}

然后训练代码都从 config 里取：

hidden_size = config["hidden_size"]
lr = config["lr"]
sample_size = config["sample_size"]
epochs = config["epochs"]

这样你记录时也不会漏。

---
5. 评估函数也要固定

建议你用统一的评估函数：

def eval_samples(samples, encoder, decoder):
    encoder.eval()
    decoder.eval()

    correct_count = 0

    for input_str, target_str in samples:
        pred = generate(input_str, encoder, decoder)
        if pred == target_str:
            correct_count += 1

    total = len(samples)
    acc = correct_count / total

    return correct_count, total, acc

这样每次实验的 train_acc / test_acc 是可比较的。

---
6. 每次训练完记录一条实验

假设你训练完之后有这些变量：

avg_epoch_loss
samples
test_samples
encoder
decoder

可以这样记录：

run_id = datetime.now().strftime("%Y%m%d_%H%M%S")

train_correct, train_total, train_acc = eval_samples(samples, encoder, decoder)
test_correct, test_total, test_acc = eval_samples(test_samples, encoder, decoder)

checkpoint_path = f"checkpoints/{run_id}_{config['model_type']}_h{config['hidden_size']}_lr{config['lr']}.pt"

record = {
    "run_id": run_id,
    "date": datetime.now().strftime("%Y-%m-%d %H:%M:%S"),
    "model_type": config["model_type"],
    "hidden_size": config["hidden_size"],
    "lr": config["lr"],
    "sample_size": config["sample_size"],
    "epochs": config["epochs"],
    "dropout": config["dropout"],
    "optimizer": config["optimizer"],
    "final_loss": avg_epoch_loss,
    "train_correct": train_correct,
    "train_total": train_total,
    "train_acc": train_acc,
    "test_correct": test_correct,
    "test_total": test_total,
    "test_acc": test_acc,
    "checkpoint_path": checkpoint_path,
    "notes": "RNN 能记住部分训练集，但测试集泛化差",
}

append_experiment_log("experiments.csv", record)

这样 experiments.csv 里就会多一行。

7. 保存 checkpoint 时也把 config 和结果放进去

权重文件里也建议保存实验元信息：

def save_checkpoint(path, encoder, decoder, config, metrics):
    checkpoint = {
        "encoder_state_dict": encoder.state_dict(),
        "decoder_state_dict": decoder.state_dict(),
        "config": config,
        "metrics": metrics,
        "vocab_size": vocab_size,
        "stoi": stoi,
        "itos": itos,
    }

    torch.save(checkpoint, path)

调用：

metrics = {
    "final_loss": avg_epoch_loss,
    "train_correct": train_correct,
    "train_total": train_total,
    "train_acc": train_acc,
    "test_correct": test_correct,
    "test_total": test_total,
    "test_acc": test_acc,
}

save_checkpoint(checkpoint_path, encoder, decoder, config, metrics)

这样以后你拿到一个 .pt 文件，也能知道：

它是什么模型？
hidden_size 是多少？
lr 是多少？
训练了多少 epoch？
测试准确率是多少？

9. 记录固定样例输出

除了数字指标，我还建议记录几个固定样例的输出。

比如：

fixed_samples = [
    ("2002-1-23", "23/1/2002"),
    ("1999-12-8", "8/12/1999"),
    ("2020-3-7", "7/3/2020"),
    ("1950-11-28", "28/11/1950"),
    ("2048-6-15", "15/6/2048"),
]

定义：

def qualitative_eval(samples, encoder, decoder):
    results = []

    encoder.eval()
    decoder.eval()

    for input_str, target_str in samples:
        pred = generate(input_str, encoder, decoder)
        results.append({
            "input": input_str,
            "target": target_str,
            "pred": pred,
            "correct": pred == target_str,
        })

    return results

然后：

fixed_results = qualitative_eval(fixed_samples, encoder, decoder)

可以把它保存到 checkpoint：

metrics = {
    "final_loss": avg_epoch_loss,
    "train_acc": train_acc,
    "test_acc": test_acc,
    "fixed_results": fixed_results,
}

如果写 CSV 不方便存复杂结构，可以把固定样例结果简单转成字符串：

notes = str(fixed_results)

或者单独保存 JSON。

10. 一次完整实验的推荐模板

你可以把每次实验流程固定成这样：

# 1. config
config = {
    "model_type": "RNN",
    "hidden_size": 128,
    "lr": 0.001,
    "sample_size": 100,
    "epochs": 1000,
    "dropout": 0.0,
    "optimizer": "Adam",
    "test_size": 100,
}

run_id = datetime.now().strftime("%Y%m%d_%H%M%S")

# 2. data
inputs, targets = make_sample(config["sample_size"])
samples = list(zip(inputs, targets))

test_inputs, test_targets = make_sample(config["test_size"])
test_samples = list(zip(test_inputs, test_targets))

# 3. model
encoder = EncoderRNN(vocab_size, config["hidden_size"])
decoder = DecoderRNN(vocab_size, config["hidden_size"], vocab_size)

encoder.dropout.p = config["dropout"]

encoder_optimizer = torch.optim.Adam(encoder.parameters(), lr=config["lr"])
decoder_optimizer = torch.optim.Adam(decoder.parameters(), lr=config["lr"])

criterion = nn.CrossEntropyLoss()

# 4. train
for j in range(config["epochs"]):
    encoder.train()
    decoder.train()

    random.shuffle(samples)
    epoch_loss = 0

    for input_str, target_str in samples:
        step_loss = train_batch_samples(
            input_str,
            target_str,
            encoder,
            decoder,
            encoder_optimizer,
            decoder_optimizer,
            criterion,
        )
        epoch_loss += step_loss

    avg_epoch_loss = epoch_loss / len(samples)

    if (j + 1) % 50 == 0:
        print("epoch:", j + 1, "avg_epoch_loss:", avg_epoch_loss)

# 5. eval
train_correct, train_total, train_acc = eval_samples(samples, encoder, decoder)
test_correct, test_total, test_acc = eval_samples(test_samples, encoder, decoder)

# 6. save checkpoint
Path("checkpoints").mkdir(exist_ok=True)

checkpoint_path = f"checkpoints/{run_id}_{config['model_type']}_h{config['hidden_size']}_lr{config['lr']}.pt"

metrics = {
    "final_loss": avg_epoch_loss,
    "train_correct": train_correct,
    "train_total": train_total,
    "train_acc": train_acc,
    "test_correct": test_correct,
    "test_total": test_total,
    "test_acc": test_acc,
}

save_checkpoint(checkpoint_path, encoder, decoder, config, metrics)

# 7. log
record = {
    "run_id": run_id,
    "date": datetime.now().strftime("%Y-%m-%d %H:%M:%S"),
    "model_type": config["model_type"],
    "hidden_size": config["hidden_size"],
    "lr": config["lr"],
    "sample_size": config["sample_size"],
    "epochs": config["epochs"],
    "dropout": config["dropout"],
    "optimizer": config["optimizer"],
    "final_loss": avg_epoch_loss,
    "train_correct": train_correct,
    "train_total": train_total,
    "train_acc": train_acc,
    "test_correct": test_correct,
    "test_total": test_total,
    "test_acc": test_acc,
    "checkpoint_path": checkpoint_path,
    "notes": "",
}

append_experiment_log("experiments.csv", record)

print(record)

11. 实验记录的核心原则

原则一：一次只改一个关键变量

不要同时改：

hidden_size
lr
epoch
sample_size
RNN/GRU
dropout

否则你不知道结果变化来自哪里。

比较推荐：

实验 1：RNN, hidden=128, lr=0.001
实验 2：RNN, hidden=256, lr=0.001
实验 3：GRU, hidden=128, lr=0.001
实验 4：GRU, hidden=256, lr=0.001

---
原则二：固定测试集

如果每次测试集都是随机生成的，实验之间会有噪声。

建议一开始生成一个固定测试集：

test_inputs, test_targets = make_sample(200)
fixed_test_samples = list(zip(test_inputs, test_targets))

后面每次实验都用同一个：

test_correct, test_total, test_acc = eval_samples(fixed_test_samples, encoder, decoder)

这样不同实验的 test_acc 才更可比。

---
原则三：记录失败实验

失败实验也要记录。

比如：

RNN hidden=128 sample=100 epoch=1000
train_acc=0.87
test_acc=0.0
结论：记忆训练集，但不泛化

这非常有价值，因为它告诉你：

不要继续在这个方向浪费时间

---
12. 什么时候需要 TensorBoard / WandB？

现在不急。

你现在阶段建议：

CSV + checkpoint metadata

就够了。

等后面实验变多，比如几十组、几百组，才考虑：

TensorBoard
Weights & Biases
MLflow

这些工具更强，但也更重。你现在用 CSV 反而最高效。

---
一句话总结

你可以把每次训练当成一条实验记录：

config + final_loss + train_acc + test_acc + checkpoint_path + notes

最小可用方案是：

用 config 管参数
用 eval_samples 统一评估
用 torch.save 保存权重和元信息
用 experiments.csv 追加记录每次实验

这样你后面就能清楚回答：

哪组参数最好？
RNN 和 GRU 谁更好？
hidden_size 增大有没有用？
lr=0.001 是否比 0.01 稳？
模型是记忆还是泛化？
