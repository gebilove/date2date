训练 loss 很低，但模型没有学到可泛化的规则，只是记住了训练集模式。

你现在的现象非常典型：
- avg_epoch_loss = 0.0298 看起来很漂亮
- 但 test 集完全错
- 生成结果里还会冒出一些训练集中常见的年份/数字

---
为什么会这样

1) 训练集太小，基本只能记忆

你只用了 100 条样本 训练。

对这种任务，100 条太少了。
模型很容易把训练样本“背下来”，但不会真正学会：

- year -> year
- month/day 位置交换
- 什么时候输出 /

所以在 train 上还能凑出不少正确结果，到了 test 就崩。

---
2) 你用的是最朴素的 encoder-decoder RNN

现在结构是：

- nn.RNN
- 没有 attention
- 没有 LSTM/GRU
- 没有显式对齐机制

这类模型把整个输入压到一个 hidden state 里。
对于“日期重排”这种需要精确复制和重排字符的任务，这种结构很弱。

---
3) 输入/输出是变长的，难度更高

你现在的样本像这样：

input : 1994-4-6
target: 6/4/1994

和：

input : 2041-10-27
target: 27/10/2041

长度变化很大。
模型不仅要学“字符搬运”，还要学：

- day 是 1 位还是 2 位
- month 是 1 位还是 2 位
- / 什么时候插入

这会明显增加学习难度。

---
4) teacher forcing 让训练 loss 看起来很好，但推理时会失真

训练时你喂的是正确答案：

input_token = target_tensor[:, i].unsqueeze(1)

所以模型每一步都在“看标准答案”。
但生成时它只能看自己上一步预测出来的 token。

这会导致：

- train loss 低
- inference 误差累积
- test 输出完全跑偏

---
5) exact match 太严格

日期任务是序列任务。
只要有一个字符错，整条样本就算错。

所以你看到的“完全不对”，有时其实是：
- 只错了一位
- 或者 month/day 顺序没学稳
- 但 sequence-level accuracy 就直接是 0

---
你代码里还有一个评估 bug

你前面这个函数：

return correct_count, correct_count/ len(fixed_samples)

应该改成：

return correct_count, correct_count / len(samples)

否则训练准确率是假的。
你之前看到的 17.4 就是这个 bug 导致的。

---
我建议你先做这几个修改

改 1：把日期改成固定宽度

这样更容易学。

def make_sample(sample_size):
    inputs = []
    targets = []
    for _ in range(sample_size):
        year = random.randint(1950, 2050)
        day = random.randint(1, 28)
        month = random.randint(1, 12)

        input = f"{year}-{month:02d}-{day:02d}"
        target = f"{day:02d}/{month:02d}/{year}"

        inputs.append(input)
        targets.append(target)
    return inputs, targets

例如：

- 2022-02-24
- 24/02/2022

这比可变长度形式更适合先验证模型能力。

---
改 2：给 encoder 输入也加 EOS

现在 encoder 输入没有 EOS。可以改成：

input_tensor = tensor_from_string(input_str, add_eos=True)

这样模型知道源序列结束了。

---
改 3：增加训练数据量

至少上千，最好上万。

比如：

inputs, targets = make_sample(20000)

然后再拆 train/test。

---
改 4：换 GRU/LSTM 或加 attention

如果你想让它真正学会“复制+重排”，建议至少换成：

- nn.GRU
- 或 nn.LSTM

更进一步加 attention 会更稳。

---
改 5：检查 token-level 准确率

不要只看 exact match。
可以先看每个字符的正确率，判断模型到底学到了多少。
